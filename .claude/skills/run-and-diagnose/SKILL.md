## 测试执行与诊断流程

1. 识别项目测试框架和运行命令：
   - 检查 package.json / pyproject.toml / Makefile / CI配置
   - 确定测试运行命令（npm test / pytest / go test 等）

2. 执行测试：
   - 运行完整测试套件或指定范围的测试
   - 收集测试结果、覆盖率报告
   - 记录执行时间和资源消耗

3. 分析失败用例：
   - 分类失败原因：
     - 代码逻辑缺陷（真正的bug）
     - 测试本身过时需要更新
     - 环境/依赖问题
     - 时序相关的不稳定测试（flaky test）
   - 对每个失败提供：根因分析 + 修复建议

4. 输出报告：
   ```
   ## 测试执行报告
   - 总数: X | 通过: X | 失败: X | 跳过: X
   - 覆盖率: X%
   ### 失败用例分析
   - [测试名] 原因: ... | 修复建议: ...
   ### 覆盖率盲区
   - [文件/函数] 未覆盖的关键路径: ...
   ```
