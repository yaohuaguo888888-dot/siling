## 测试编写流程

1. 分析被测代码：
   - 识别公共接口和关键方法
   - 梳理输入参数的有效范围和边界值
   - 识别外部依赖需要mock的部分
   - 列出所有代码分支和执行路径

2. 设计测试用例矩阵：
   - 正常路径：典型输入 → 预期输出
   - 边界条件：空值、零值、最大值、最小值
   - 异常路径：无效输入、网络错误、超时
   - 状态转换：前置条件 → 操作 → 后置条件

3. 编写测试代码：
   - 遵循 Arrange-Act-Assert 模式
   - 测试函数命名：`test_<被测行为>_<条件>_<预期结果>`
   - 使用参数化测试减少重复代码
   - 合理设置setup/teardown

4. 验证测试质量：
   - 确保测试能在被测代码有bug时失败
   - 检查是否有遗漏的重要路径
   - 确认mock行为与真实依赖一致
