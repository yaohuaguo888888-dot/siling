## 代码生成流程

1. 仔细阅读需求描述/Issue内容，提取功能要求和约束条件
2. 分析现有代码库结构（如有），确定：
   - 使用的技术栈和框架
   - 项目的目录组织规范
   - 已有的工具函数和公共模块
   - 代码风格（命名规范、缩进、注释风格）
3. 设计实现方案：
   - 模块划分和文件结构
   - 核心接口和数据结构定义
   - 依赖关系和调用链路
4. 编写代码时遵循原则：
   - 单一职责，每个函数/类只做一件事
   - 防御性编程，处理边界条件和错误情况
   - 添加必要的类型注解和文档注释
   - 预留可测试性（依赖注入、纯函数优先）
5. 生成完整的文件清单和代码内容
6. 附带简要的设计说明文档
