---

## 架构合规检查清单

### 分层规范检查
- [ ] 表现层是否绕过了业务逻辑层直接访问数据层？
- [ ] 数据层是否包含了业务逻辑？
- [ ] 是否存在循环依赖？
- [ ] 模块间通信是否通过约定的接口/协议？

### 设计模式一致性
- [ ] 新代码是否遵循项目已有的设计模式？
- [ ] 是否引入了与现有模式冲突的新模式？
- [ ] 单例/工厂/观察者等模式使用是否恰当？

### 依赖管理
- [ ] 新增依赖是否必要且合理？
- [ ] 依赖版本是否与项目策略一致？
- [ ] 是否引入了过重的依赖（可用轻量替代）？

### 可维护性
- [ ] 代码组织是否便于未来扩展？
- [ ] 是否违反了开闭原则？
- [ ] 硬编码的配置是否应抽取为配置项？

### 输出
对每个违规项说明：违反了什么原则、具体位置、建议的修正方式。
